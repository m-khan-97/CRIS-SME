Public exposure assessment

Authorised surface analysis

Scoped passive metadata collection only

CRIS-SME collects DNS A/AAAA records, HTTP response codes, HTTPS availability, TLS certificate metadata, and security-header presence for explicitly entered targets. No exploitation, brute force, crawling, port sweeping, or internet-wide scanning is performed.

Target scope

One hostname or IP per line

Evidence collected per target

Passive control-plane signals
DNS A / AAAA resolution

IPv4 and IPv6 addresses observed for the entered hostname. Used to confirm the target resolves and to check for unexpected IP ranges.

HTTP / HTTPS Response code and redirect chain

HTTP status code, redirect-to-HTTPS behaviour, and final response status. HTTPS availability is a CE and UK GDPR signal.

TLS Certificate and protocol version

TLS protocol version, certificate validity, expiry date, and issuer. Expired or weak TLS configurations surface as CE control gaps.

Security headers HSTS, CSP, X-Frame-Options

Presence or absence of security-relevant HTTP response headers. Missing headers may indicate misconfigured web surface.
